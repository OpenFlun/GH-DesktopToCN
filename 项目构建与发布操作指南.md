# GitHubDesktopToCN 项目构建与发布操作指南

## 一、前置准备（仅第一次需要）

这些是一次性配置,之后无需再动；

| 事项                                 | 路径/操作                                                                                 | 注意点                                                         |
| ------------------------------------ | ----------------------------------------------------------------------------------------- | -------------------------------------------------------------- |
| 1. 安装 Qt6                          | `C:\Qt\6.11.0\msvc2022_64`                                                                | 路径必须与 `release.bat` 中的 `QT_DIR` 一致                    |
| 2. 安装 Enigma Virtual Box           | 任意路径                                                                                  | `release.bat` 中 `ENIGMA_CONSOLE` 要指向 `enigmavbconsole.exe` |
| 3. 安装 GitHub CLI                   | `C:\Program Files\GitHub CLI\gh.exe`                                                      | 需提前用 `gh auth login` 登录认证                              |
| 4. 安装 Visual Studio 或 Build Tools | `C:\Program Files (x86)\Microsoft Visual Studio\18\BuildTools\Common7\Tools\VsDevCmd.bat` | 脚本通过 VsDevCmd.bat 加载 MSVC 环境                           |

### **⭐ 最重要的一步：创建封包模板**
1. 打开 Enigma Virtual Box **图形界面**（不是控制台）；
2. **输入文件**：随便选一个 exe（例如 `C:\Windows\system32\notepad.exe`）,稍后会被覆盖,不重要；
3. **输出文件**：随便填,如 `D:\boxed.exe`；
4. 点击“**添加文件夹**”,路径选择你的安装目录（**注意：此时安装目录可能还不存在,可先创建空目录占位,模板只要路径结构正确即可**）,勾选“包含子文件夹”,目标文件夹留空；
5. **请确保该目录包含 OpenSSL 动态库**（`libcrypto-3-x64.dll` 和 `libssl-3-x64.dll`）,否则打包后的 exe 将无法使用 HTTPS；
6. 菜单栏 → **文件** → **另存为**,保存到项目根目录：
   ```
   D:\E-lang\GH-DesktopToCN\release_template.evb
   ```
7. 关闭 GUI；

> **致命警告**：`enigmavbconsole.exe` **无法解析任何手写的 XML 或 INI 文本**,必须使用 GUI 生成的二进制 `.evb` 文件；如果你跳过这一步,封包步骤必然失败并报 `Cannot open file ""`；此模板只需创建一次,永远有效;

## 二、日常构建与发布流程

只需**双击运行 `release.bat`**,脚本会自动按顺序执行：

| 步骤                   | 说明                                                               | 输出                                                       |
| ---------------------- | ------------------------------------------------------------------ | ---------------------------------------------------------- |
| `[1/6]` 加载 MSVC 环境 | 调用 `VsDevCmd.bat` 设置编译环境                                   |                                                            |
| `[2/6]` 构建 Release   | 使用 CMake presets 编译,若代码未改动则显示 `ninja: no work to do.` | `out/build/x64-release/` 下的产物                          |
| `[3/6]` 安装           | 将 exe 拷贝到干净目录                                              | `out/install/x64-release/GitHubDesktopToCN.exe`            |
| `[4/6]` 部署 Qt        | `windeployqt` 自动复制所需 DLL 和插件,**并复制 OpenSSL DLL**       | 安装目录下出现 `Qt6Core.dll`、`libcrypto-3-x64.dll` 等文件 |
| `[5/6]` 封包单文件     | 使用 `release_template.evb` 将整个安装目录打包为单一 exe           | `release/GitHubDesktopToCN.exe`                            |
| `[6/6]` 同步版本号     | 从 `CHANGELOG.md` 提取版本号并写入 `version.json`                  | `version.json` 中的版本更新（由脚本自动覆盖）              |
| `[7/7]` 发布到 GitHub  | 创建 Release 草稿并上传 exe 和 `localization.json`                 | GitHub Releases 页面出现新草稿                             |

**如果希望发布正式版（公开可见）**：在 GitHub 网页上找到刚创建的草稿,点击“Publish release”按钮即可；
或者修改 `release.bat`,删除 `gh release create` 命令中的 `--draft` 参数,使其直接公开发布;

## 三、项目文件说明

| 文件                   | 作用                                                                                                     |
| ---------------------- | -------------------------------------------------------------------------------------------------------- |
| `CMakeLists.txt`       | 构建配置,定义预汉化资源嵌入、**OpenSSL 头文件与导入库链接**、Qt 链接                                     |
| `CMakePresets.json`    | CMake x64 Debug/Release 预设                                                                             |
| `release.bat`          | 一键构建、打包、发布的自动化脚本                                                                         |
| `release_template.evb` | Enigma Virtual Box 封包模板（**必须由 GUI 生成**）                                                       |
| `version.json`         | 当前版本号,格式 `{"version":"1.0.0"}`,**由 `release.bat` 从 `CHANGELOG.md` 自动提取并写入,无需手动修改** |
| `prebuilt/`            | 汉化资源目录,包括 `localization.json`,构建时会被编译进 exe **并同时作为独立文件上传到 Release**          |
| `resources.qrc`        | 自动生成的 Qt 资源文件,无需手动修改                                                                      |

## 四、发布后的访问地址

以账户 `OpenFlun`、仓库 `GH-DesktopToCN`、版本 `1.0.1` 为例：

*   **Release 页面**（查看所有文件和说明）：
    ```
    https://github.com/OpenFlun/GH-DesktopToCN/releases/tag/1.0.1
    ```
*   **直接下载 exe**：
    ```
    https://github.com/OpenFlun/GH-DesktopToCN/releases/download/v1.0.1/GitHubDesktopToCN.exe
    ```
*   **直接下载 localization.json**：
    ```
    https://github.com/OpenFlun/GH-DesktopToCN/releases/download/v1.0.1/localization.json
    ```
*   **始终指向最新 Release 的链接**（推荐写在 README 中）：
    ```
    # 最新页面
    https://github.com/OpenFlun/GH-DesktopToCN/releases/latest
    # 最新 exe 直链
    https://github.com/OpenFlun/GH-DesktopToCN/releases/latest/download/GitHubDesktopToCN.exe
    ```

> **关键注意**：下载直链中的版本号 **必须加 `v` 前缀**（即 `v1.0.1`）,哪怕你的版本号本身没有 `v`；这是 GitHub 的固定规则,写错将 404;

## 五、常见问题及处理

### 1. 版本号想跳大版本（例如 1.0.x → 1.1.0）
直接修改 `CHANGELOG.md` 中第一个版本标题的版本号（例如 `## [1.1.0] - 2026-05-14`）,脚本会自动使用该版本号并同步到 `version.json`;

### 2. 程序新增了 DLL 或插件依赖
重新运行一次 Enigma Virtual Box GUI,打开 `release_template.evb`,删除旧的文件夹路径,重新添加安装目录（确保勾选“包含子文件夹”）,保存并覆盖模板;

### 3. 构建时提示 `ninja: no work to do.` 但代码的确改了
可能是 CMake 缓存导致,删除 `out/build/x64-release` 目录后重新运行 `release.bat`;

### 4. 上传失败：GitHub CLI 未登录
在命令行手动执行 `gh auth login`,按提示完成认证;

### 5. 封包失败：`Cannot open file ""`
你忘了创建或丢失了 `release_template.evb`；请重新用 GUI 生成模板并放到项目根目录;

### 6. 封包后的 exe 运行时提示“找不到 libcrypto-3-x64.dll”或网络请求失败
- **原因**：封包模板创建时,安装目录中没有包含 OpenSSL 动态库；或 `release.bat` 中复制 DLL 的步骤失败;
- **解决**：
  - 检查 `openssl-proj\lib` 目录下是否存在 `libcrypto-3-x64.dll` 和 `libssl-3-x64.dll`;
  - 手动运行一次 `release.bat`,观察 `[4/6]` 步骤是否输出“复制 xxx.dll”;
  - 重新用 GUI 打开 `release_template.evb`,删除旧文件夹添加,重新选择包含 DLL 的安装目录,保存模板后重新发布;

## 六、附件：release.bat 完整脚本

```bat
@echo off
setlocal enabledelayedexpansion

:: ==================== 配置 ====================
set "PROJECT_DIR=%~dp0"
if "%PROJECT_DIR:~-1%"=="\" set "PROJECT_DIR=%PROJECT_DIR:~0,-1%"

set "QT_DIR=C:\Qt\6.11.0\msvc2022_64"
set "OPENSSL_DLL_DIR=%PROJECT_DIR%\openssl-proj\lib"
set "ENIGMA_CONSOLE=D:\RuanJian\Enigma Virtual Box\enigmavbconsole.exe"
set "VSDEVCMD=C:\Program Files (x86)\Microsoft Visual Studio\18\BuildTools\Common7\Tools\VsDevCmd.bat"
set "GH_EXE=C:\Program Files\GitHub CLI\gh.exe"

set "BUILD_DIR=%PROJECT_DIR%\out\build\x64-release"
set "INSTALL_DIR=%PROJECT_DIR%\out\install\x64-release"
set "SINGLE_EXE=%PROJECT_DIR%\release\GitHubDesktopToCN.exe"
set "RELEASE_DIR=%PROJECT_DIR%\release"

:: ==================== 0. 环境检查 ====================
if not exist "%GH_EXE%" ( echo 错误: GitHub CLI 未找到 & exit /b 1 )
if not exist "%VSDEVCMD%" ( echo 错误: VsDevCmd.bat 未找到 & exit /b 1 )
if not exist "%QT_DIR%" ( echo 错误: Qt 目录不存在 & exit /b 1 )
if not exist "%ENIGMA_CONSOLE%" ( echo 错误: enigmavbconsole.exe 未找到 & exit /b 1 )

:: ==================== 1. 加载 MSVC 环境 ====================
echo [1/6] 加载 MSVC 编译器环境...
call "%VSDEVCMD%" -arch=amd64
if errorlevel 1 ( echo 环境加载失败！ & exit /b 1 )

:: ==================== 2. 从 CHANGELOG.md 提取版本号 ====================
echo [2/6] 从 CHANGELOG.md 提取版本号...
set "CHANGELOG_FILE=%PROJECT_DIR%\CHANGELOG.md"

if not exist "%CHANGELOG_FILE%" (
    echo 错误: 未找到 CHANGELOG.md
    exit /b 1
)

:: 提取第一个版本号（不生成 Release Notes 文件）
set "start="
for /f "tokens=1,* delims=:" %%i in ('findstr /n /c:"## [" "%CHANGELOG_FILE%" ^| findstr /v /c:"https"') do (
    if not defined start (
        set "start=%%i"
        set "verline=%%j"
        set "verline=!verline:*## [=!"
        set "verline=!verline:]=!"
        for /f "tokens=1 delims= " %%v in ("!verline!") do set "VERSION=%%v"
        goto :found_version
    )
)
:found_version
if not defined start (
    echo 错误: 未在 CHANGELOG.md 中找到版本条目
    exit /b 1
)

:: ==================== 3. 版本号检查 ====================
echo [3/6] 检查版本号是否大于上次发布版本...
set "OLD_VERSION_FILE=%PROJECT_DIR%\version.json"
set "OLD_VERSION="

if exist "%OLD_VERSION_FILE%" (
    for /f "usebackq tokens=2 delims=:," %%a in (`findstr /c:"\"version\"" "%OLD_VERSION_FILE%"`) do (
        set "OLD_VERSION=%%~a"
    )
    if defined OLD_VERSION (
        set "OLD_VERSION=!OLD_VERSION:"=!"
        set "OLD_VERSION=!OLD_VERSION: =!"
        set "OLD_VERSION=!OLD_VERSION:{=!"
        set "OLD_VERSION=!OLD_VERSION:}=!"
        set "OLD_VERSION=!OLD_VERSION:,=!"
        for /f "tokens=1 delims= " %%v in ("!OLD_VERSION!") do set "OLD_VERSION=%%v"
    )
)

if defined OLD_VERSION (
    :: 使用 PowerShell 进行版本比较
    powershell -Command "& { if ([version]'%VERSION%' -gt [version]'!OLD_VERSION!') { exit 0 } else { exit 1 } }" >nul 2>&1
    if errorlevel 1 (
        echo 错误: 待发布版本 %VERSION% 应大于旧版本 !OLD_VERSION!
        pause >nul
        exit /b 1
    )
    echo 版本验证通过: 待发布版本 %VERSION% !
) else (
    echo 未找到 version.json,首次发布,跳过版本比较;
)

:: ==================== 4. 远程检查（避免已发布）====================
echo [4/6] 检查 GitHub 上是否已存在同名 Release...
"%GH_EXE%" release view "%VERSION%" >nul 2>&1
if not errorlevel 1 (
    echo 错误: Release %VERSION% 已存在于 GitHub 上,请更新 CHANGELOG.md 中的版本号后再发布;
    exit /b 1
)

:: ==================== 5. 生成 Release Notes 文件 ====================
echo [5/6] 生成 Release Notes...
set "RELEASE_NOTES_FILE=%TEMP%\release_notes_%RANDOM%.md"

:: 找到下一个版本标题的行号
set "end="
for /f "tokens=1 delims=:" %%i in ('findstr /n /c:"## [" "%CHANGELOG_FILE%" ^| findstr /v /c:"https"') do (
    if %%i gtr !start! (
        set "end=%%i"
        goto :found_end
    )
)
:found_end
if not defined end set "end="

:: 提取内容并写入 Release Notes 文件（跳过标题行）
del "%RELEASE_NOTES_FILE%" 2>nul
for /f "tokens=1,* delims=:" %%i in ('findstr /n "^" "%CHANGELOG_FILE%"') do (
    if %%i gtr !start! (
        if defined end (
            if %%i geq !end! goto :done_notes
        )
        if %%i neq !start! (
            echo(%%j >> "%RELEASE_NOTES_FILE%"
        )
    )
)
:done_notes
echo Release Notes 已保存到: %RELEASE_NOTES_FILE%

:: 同步更新 version.json（保持版本号一致）
echo { "version": "%VERSION%" } > "%PROJECT_DIR%\version.json"

:: ==================== 6. 构建 Release ====================
echo [6/6] 开始构建 Release 版本（版本 %VERSION%）...
if not exist "%BUILD_DIR%\CMakeCache.txt" (
    echo 首次配置...
    cmake -S "%PROJECT_DIR%" --preset x64-release -DCMAKE_PREFIX_PATH="%QT_DIR%" -DGitHubActions=ON -DFILE_VERSION=%VERSION%
    if errorlevel 1 ( echo 配置失败！ & exit /b 1 )
)
cmake --build "%BUILD_DIR%" --config Release
if errorlevel 1 ( echo 构建失败！ & exit /b 1 )

:: ==================== 7. 安装 ====================
echo [7/7] 安装到干净目录...
cmake --install "%BUILD_DIR%" --config Release --prefix "%INSTALL_DIR%"
if errorlevel 1 ( echo 安装失败！ & exit /b 1 )

:: ==================== 8. 部署 Qt 与 OpenSSL DLL ====================
echo 部署 Qt 运行库...
cd /d "%INSTALL_DIR%"

if exist "%OPENSSL_DLL_DIR%" (
    for %%F in ("%OPENSSL_DLL_DIR%\libcrypto*.dll" "%OPENSSL_DLL_DIR%\libssl*.dll") do (
        if exist "%%F" (
            echo 复制 %%~nxF ...
            copy /y "%%F" . >nul
        )
    )
) else (
    echo 警告: 未找到 OpenSSL DLL 目录 %OPENSSL_DLL_DIR%
)

call "%QT_DIR%\bin\windeployqt.exe" "GitHubDesktopToCN.exe" --no-translations --no-system-d3d-compiler --compiler-runtime --force-openssl
if errorlevel 1 ( echo windeployqt 失败！ & cd /d "%PROJECT_DIR%" & exit /b 1 )
cd /d "%PROJECT_DIR%"

:: ==================== 9. 封包成单一 exe ====================
echo 打包成单文件 exe...
if not exist "%RELEASE_DIR%" mkdir "%RELEASE_DIR%"

if not exist "%PROJECT_DIR%\release_template.evb" (
    echo 错误：未找到模板文件 "%PROJECT_DIR%\release_template.evb"
    exit /b 1
)

"%ENIGMA_CONSOLE%" "%PROJECT_DIR%\release_template.evb" -input "%INSTALL_DIR%\GitHubDesktopToCN.exe" -output "%SINGLE_EXE%"
if %errorlevel% neq 0 (
    echo 封包失败！错误代码: %errorlevel%
    exit /b 1
)

del "%TEMP%\enigma_project.evb" >nul 2>&1

if not exist "%SINGLE_EXE%" (
    echo 错误：单文件 exe 未生成！
    exit /b 1
)
echo 单文件 exe 已生成: %SINGLE_EXE%

:: ==================== 10. 发布到 GitHub ====================
echo 正在创建 GitHub Release...
"%GH_EXE%" release create "%VERSION%" --title "%VERSION%" --notes-file "%RELEASE_NOTES_FILE%"
if errorlevel 1 (
    echo 创建 Release 失败！
    del "%RELEASE_NOTES_FILE%" 2>nul
    exit /b 1
)

echo 正在上传单文件 exe...
"%GH_EXE%" release upload "%VERSION%" "%SINGLE_EXE%" --clobber
if errorlevel 1 (
    echo 上传 exe 失败！
    del "%RELEASE_NOTES_FILE%" 2>nul
    exit /b 1
)

set "JSON_FILE=%PROJECT_DIR%\prebuilt\localization.json"
if exist "%JSON_FILE%" (
    echo 正在上传 localization.json ...
    "%GH_EXE%" release upload "%VERSION%" "%JSON_FILE%" --clobber
    if errorlevel 1 (
        echo 上传 localization.json 失败！
        del "%RELEASE_NOTES_FILE%" 2>nul
        exit /b 1
    )
)

del "%RELEASE_NOTES_FILE%" 2>nul

echo ==============================
echo   发布完成！版本 %VERSION%
echo ==============================
cd /d "%PROJECT_DIR%"
endlocal
exit /b 0
```

这份指南应该能覆盖你后续所有的发布场景；如果有需要调整的地方,直接修改 `release.bat` 中对应的变量即可;祝发布顺利！
